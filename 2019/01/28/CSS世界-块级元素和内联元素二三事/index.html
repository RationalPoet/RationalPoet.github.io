CSS世界-块级元素和内联元素二三事

YoungSure 1月 28, 2019
  • CSS
  • 分享到微博
  • 分享到 LinkedIn

序

CSS世界中的元素分为了块级元素（block-level element）和内联级元素（inline-level element）。其中内联级元素是在W3C在CSS规范中的叫法，其实W3C在HTML4标准中明确地将HTML元素分为了“块级元素”和“内联元素（inline element）”，虽然叫法上有些出入，但其实本质上是同一个东西。

既然CSS世界中的元素是由块级元素和内联元素组成的，那么我们就在这里先简单地逐个介绍下块级元素和内联元素，这里会分享一些大家平时很少关注的元素特性，也会对其做一个总结描述，希望大家可以更全面细致地去区分块级元素与内联元素之间的玄妙关系。

块级元素

常见的块级元素有div、p、li、table等元素。这是需要注意的是：并不是所有“display属性是block”的元素都是“块级元素”，其根本就不是一个概念。例如：li元素的”display”属性是”list-item”，table元素的”display”属性是”table”，但是它们是块级元素是确定无疑的。因为它们都符合块级元素的基本特征：“块级元素都是独占一行展示的，多个块级元素需要换行展示”！

这里延伸一下，由于“块级元素”的独占一行、换行展示的特性，我们经常把它和“clear”属性配合使用来清除浮动带来的影响， 如：
.clear:after { content: '', display: block, clear: both }

在我们平时的开发过程中，经常会想使用块级元素的一些块级特性来应用在内联元素上面，我们的做法通常是把内联元素的”display”属性重置为”block”或”table”，很少有人会想到重置为”list-item”吧，那么如果我们果真把”display”属性重置为”list-item”会怎样呢？
结果大家试一下便知道了，就是在元素的前面多了一个“项目符号”，虽然我们可以设置”list-style”属性的属性值为”none”来解决这个问题，但是，大家有没有想过为什么设置了”display”属性为”list-item”之后，元素符号会出现项目符号呢？

为什么list-item元素会出现项目符号

在CSS世界伊始，其目的很单纯（CSS2及之前）的，只是单纯的做图文编排，一切布局特性都要以“更好地实现图文编排”为核心。所以，你会发现在CSS2标准中有很多属性的展示特性甚至“bug”的出现都是为了更好的实现图文编排而被“精心设计出来的”。

最初，CSS世界只存在“块级盒子”和“内联盒子”，块级盒子负责结构布局，内联盒子负责内容展示。

但是谁曾想，有些特殊的文字编排需求致使了一些特殊的需求：我在展示列表类型数据的时候，系统能不能自己给我的每项列表之前加上一个符号，让我们的读者能轻易区分开每项数据呢？于是，”list-item”便由此诞生了。

标记盒子

元素原来只有“块级盒子”和“内联盒子”，现在你多出来个”list-item”的”display”属性值，你应该叫个啥呢？W3C标准正色到，就叫“标记盒子”吧，专门用来放置圆点、数字等项目符号。

外在盒子，容器盒子

有了”list-item”的先河，元素又想特殊点了：“我能不能表面是内联元素（紧挨着内联元素展示），内地里却能拥有块级元素的一些特性呢？例如设置下宽高？”。这样的呼声越来越高，最后，在结合实际使用场景和需求的前提下，W3C又创建出了“外在盒子”和“内在盒子”。

“外在盒子”负责该元素是可以一行展示，还是要换行展示也就是，”内在盒子”则负责宽高的设置以及内容展示等。

每个元素都有两个盒子，“外在盒子”和“内在盒子”。但是“内在盒子”比较晦涩，后来改名称为“容器盒子”。

所以，按照”display”属性值的不同，这个元素就有不同的“外在盒子”和“容器盒子”组成了：”block”的元素，是由外在的”块级外在盒子”和”块级容器盒子”组成的，”inline-block”是由外在的“内联外在盒子”和“块级容器盒子”组成，而”inline”则是由内外盒子都是“内联盒子”组成的。

width和height的作用细节

我们已经知道了，width和height是设置在“容器盒子”上的，那么width和height的作用细节又是怎么样的呢？下面我们就分析下width和height是如何作用在“容器盒子”上，致使元素表现出“千奇百怪、婀娜多姿”的一面的。

你不知道的width:auto

width的默认值就是auto。

作为width的默认值，而我们大家又很喜欢给元素设置各种各样自己需要的宽度值，所以很少用到”auto”这个值作为width的属性值，所以也就意味着”auto”隐藏了很多我们不知道的秘密：

  • 充分利用可用空间，比如div、p等块级元素,他们的默认宽度都是100%于父容器的，块级元素这种充分利用父容器宽度的特性是我们潜移默化中共同默认的一种行为，其学名叫做：fill-available，其实是块级元素在”auto”属性值下的一种特性行为；
  • 收缩性与包裹性，典型代表就是浮动、绝对定位、”inline-block”元素或”table”元素，学名叫做：shrink-to-fit,这些典型代表在没设置width属性值时，就会出现这种“收缩到合适”的特性行为；
  • 收缩到最小，最常出现在”table-layout”为”auto”的表格中。最常见的就是表格布局中的中文内容“一柱擎天”的盛况。当表格每一列空间都不足的时候，文字能断就断,但是中文是随便断的，英文单词不能断，最终中文内容很容易就出现了“一柱擎天”的现象。这种行为在规范中被称为”preferred minimum width”或”minimum content width”，后来也被称为”min-content”。
    具体示例可以参见张鑫旭老师的demo示例地址： demo地址
  • 超出容器限制，除非”width”属性有相关设置，否则以上3种情况都不会玩超出父容器宽度的，但是存在一些特殊情况。例如,内容很长的连续的英文或数字,或者内联元素被设置了white-space: nowrap,则表现为“恰似一江春水向东流，流到断崖也绝不回头”。这种现象在规范中后来被称为”max-content”
    具体示例可以参见张鑫旭老师的demo示例地址： demo地址

以上四类场景都是我们在使用width的默认值”auto”时不同元素的宽度表现。

在CSS世界中，元素有“外在盒子”和“容器盒子”，显示也分“内部展示”和“外部展示”，同样的，尺寸也分“内部尺寸（intrinsic sizing）”和“外部尺寸(extrinsic sizing)”，内部尺寸表示尺寸由内部元素决定，而外部尺寸则表示其尺寸是由外部元素决定的。

所以，上面我们所列举的四类”auto”的默认特性，只有第一种“充分利用可用空间”是“外部尺寸”，其他三种则都属于“内部尺寸”。

外部尺寸和流体特性
  1. 正常流宽度。当我们在一个容器中倒入足量的水，水一定会均匀铺满整个容器。像<div>、<p>元素，其外部尺寸特性就会像水流一样，铺满容器，这就是’block’属性外在盒子的特性。
  2. 格式化宽度。格式化宽度出现在“绝对定位模型”中，也就是出现在’position’属性为’fixed’或’absolute’的元素中。通常情况下，处于“绝对定位模型”的元素具有“包裹性”，即其外部尺寸由内容决定，但是有一种特殊情况，元素的外部尺寸是由外部决定的：
    对于非替换元素，当left/right或top/bottom成对出现时，元素的宽度表现出来“格式化宽度”的特性，其宽度大小是相对于最近的’position’属性不为static的祖先元素来计算的。
    内部尺寸和流体特性
  3. 包裹性。即’shrink-to-fit’，包裹性除了包裹还有自适应的意思。
    所谓“自适应”，指的是元素的尺寸由内部元素决定，但是永远小于“包含块”容器的尺寸（除非容器尺寸小于元素的“首选最小宽度”）。
    内部尺寸的包裹性最典型的示例就是实现：“单行文字居中，多行文本左对齐”这种需求场景了，代码如下：
    .box { text-align: center; } .content { display: inline-block; text-align: left; }
  4. 首选最小宽度。所谓首选最小宽度，指的是元素最适合的宽度。在CSS世界中（CSS2），图片和文字的权重要大于布局，因此制定了如下的首选最小宽度规则：
    • 东亚文字最小宽度为每个汉字的宽度;
    • 西方文字最小宽度由特定的连续的英文字符单元决定。并不是所有的英文字符都会组成连续单元，一般会终止于空格（普通空格）、短横线、问号以及其他非英文字符等。例如，’display:inline-block’这几个字符以连接符’-‘作为分隔符，形成了’display:inline’和’block’两个连续单元，由于连接符’-‘分隔位置在字符后面，因此，最后的宽度就是’display:inline-‘的宽度。（如果想让英文字符和中文一样，每一个字符都用最小宽度
      单元，可以试试使用 CSS 中的 word-break:break-all）
    • 类似图片这样的替换元素，其首选最小宽度就是元素内容自身的宽度。
