什么叫作用域

一套设计良好的规则来存储变量，并可以在之后方便找到这些变量，这套规则就叫做“作用域”
  1. JavaScript通常被归类为“解释执行”或“动态”语言。其实，JavaScript也是一门编译语言。只是与传统编译语言不同的是，它不是提前编译的，而是在代码执行前（几微秒甚至更短）编译完成的；
  2. 任何JavaScript代码在执行之前都要进行编译；
  3. 因为有代码“执行前编译”的存在，那么就存在编译器和执行引擎的配合工作。如下面一段代码示例，编译器生成代码后，引擎执行时会查找变量a是否声明过，这时就需要作用域的协助来进行查找，但是引擎执行怎么样的查找，会影响最终的查找结果。
  4. 作用域嵌套：LHS 和 RHS 查询都会在当前执行作用域中开始，如果有需要（也就是说它们没有找到所需的标识符） ，就会向上级作用域继续查找目标标识符，这样每次上升一级作用域（一层楼） ，最后抵达全局作用域（顶层） ，无论找到或没找到都将停止；
  5. 不成功的 RHS 引用会导致抛出 ReferenceError 异常。不成功的 LHS 引用会导致自动隐式地创建一个全局变量（非严格模式下）。

词法作用域

作用域共有两种主要的工作模型。第一种是最为普遍的，被大多数编程语言所采用的词法作用域，另外一种叫作动态作用域。而JavaScript使用的即词法作用域
  1. 词法作用域就是定义在词法阶段的作用域。即，词法作用域是由你写的代码时将变量和块作用域写在哪里决定的；
  2. 一般情况下，写代码时写定的变量和块作用域当词法分析器处理代码时不会做任何改变，然而也有特殊情况（eval和with会更改作用域）：
    • eval，可以对一段包含一个或多个声明的“代码”字符串进行演算，并借此来修改已经存在的词法作用域（在运行时）；
    • with，本质上是通过将一个对象的引用当作作用域来处理，将对象的属性当作作用域中的标识符来处理，从而创建了一个新的词法作用域（同样是在运行时）；
    • 这两个机制的副作用是引擎无法在编译时对作用域查找进行优化，因为引擎只能谨慎地认为这样的优化是无效的。使用这其中任何一个机制都将导致代码运行变慢。
  3. 作用域具有嵌套特性，且严格包含，作用域的最顶层为全局作用域：
    • 作用域内查找时，如果在当前作用域内未查找（LHS或RHS）到变量，就会向上一层作用域进行查找，查到即返回，未查到则一直到全局作用域，不管是否查到结果都结束查找；
    • 作用域在查找到第一个匹配的标识符时就停止，在多层嵌套的作用域中可以定义同名标识符，这叫做“遮蔽效应”；
    • 词法作用域查找只会查找一级标识符。

函数作用域

JavaScript中最常见的作用域气泡，通常指的是基于函数的作用域，由声明函数而形成的作用域就叫做函数作用域
  1. 由于作用域的特性（作用域气泡的嵌套），导致内层作用域可以访问外层作用域的变量，反过来会报错。这一特性会变得很有意思；
  2. 隐藏内部实现，把所写的代码片段通过函数声明对其进行包装，实际上就把这些代码隐藏掉了。这样做的好处，主要是引申了编码设计中的最小特权原则，可以做到一定的数据私密性；
  3. 规避冲突， 隐藏作用域中变量的一个好处是可以避免同名标记符之间的冲突：
    • 全局命名空间，主要是一些第三方库在使用时会暴露其变量在全局变量中，这样就存在命名冲突，甚至覆盖的风险；
    • 模块管理，使用模块管理器来管理第三方库是一个比较好的做法，然而事实上，模块管理器依然是建立在作用域“隐藏内部实现”这一神奇规则上的。
  4. 匿名和具名函数，在隐藏内部实现的时候（把变量和作用域包裹在一个函数声明里），依然会有问题，即函数名也会存在变量污染的问题。因此，我们就需要借助IIFE（立即执行函数）来做这件事情，即(function a{xxx})(),这样就可以规避掉上述问题。这边的IIFE，其中的函数可以是具名的，也可以是匿名的，但最佳实践为使用具名的IIFE，理由如下：
    • 匿名函数在栈追踪中不会显示出有意义的函数名，使得调试很困难；
    • 如果没有函数名，当函数需要引用自身时只能使用已经过期的 arguments.callee 引用，比如在递归中。另一个函数需要引用自身的例子，是在事件触发后事件监听器需要解绑自身；
    • 匿名函数省略了对于代码可读性/可理解性很重要的函数名。一个描述性的名称可以让代码不言自明。

块作用域

在ES5及其之前的标准中，JavaScript中是有块作用域的，例如我们的with，还有try-catch，但是，这些东西的块作用域只是象征性的摆在那里，对于with的性能开销，和严格模式下的禁用，我们不会考虑使用，而try-catch，只能做达到try中参数error的声明来作为块作用域，暂时意义不大。我们这边重点讲一下ES6中新增的块作用域语法
